# 项目核心概念

## 两种连接状态

### 1. 离线（Offline）
- **定义**：客户端与后端 PHP 服务器（Laravel API）之间没有建立连接
- **表现**：无法从服务器获取数据、无法同步到云端
- **降级策略**：自动使用本地 JSON 文件存储（LocalStore）
- **影响范围**：
  - 人设数据只保存在本地
  - 内容数据只保存在本地
  - 统计数据无法同步
  - AI 生成功能可能受限

### 2. 未连接（Disconnected）
- **定义**：客户端与自媒体平台（如小红书）之间没有建立连接
- **判断标准**：项目根目录下的 Cookie 文件不存在或已失效
- **表现**：无法发布内容到该平台、无法获取平台数据
- **连接方式**：通过小红书 MCP 扫码登录

## 小红书登录流程

### 正确的登录流程
1. **触发登录**：用户点击"连接小红书"按钮
2. **调用 MCP**：通过小红书 MCP 生成独立的二维码（不是浏览器页面）
3. **用户扫码**：用户使用小红书 APP 扫描二维码
4. **获取 Cookie**：扫码成功后，MCP 返回 Cookie 数据
5. **保存 Cookie**：将 Cookie 保存到项目根目录
6. **提示成功**：立即提示用户"登录成功"，不需要等待其他操作

### 登录成功的判定标准
- ✅ **唯一标准**：成功获取并保存 Cookie 文件
- ❌ **不应该**：等待获取用户主页信息
- ❌ **不应该**：等待其他网络请求完成

### 连接状态的判断
- **已连接**：Cookie 文件存在且有效
- **未连接**：Cookie 文件不存在或已失效

## 技术实现要点

### 小红书 MCP
- 位置：`tools/bin/xhs-mcp`
- 功能：管理小红书登录和 Cookie
- 二维码生成：应该是独立的二维码图片，不是浏览器窗口

### Cookie 管理
- 存储位置：项目根目录
- 验证方式：检查文件是否存在且未过期
- 更新时机：每次扫码登录成功后

## 用户体验原则

1. **快速反馈**：登录成功后立即提示，不要让用户等待
2. **清晰状态**：明确区分"离线"和"未连接"两种状态
3. **简化流程**：扫码登录应该是最简单的方式，不需要打开浏览器
