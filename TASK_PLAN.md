# 阶段2: 人设系统

## 目标
用户可以创建和管理多个IP人设，每个人设包含性格、风格、领域等设定

## 实现内容
1. 人设数据模型（已在 shared/types.ts 定义）
2. 人设服务层（CRUD操作）
3. 人设IPC Handler
4. 人设管理界面
   - 人设列表
   - 创建/编辑人设表单
   - 删除确认

# 阶段4: 内容管理（重点）

## 目标
管理服务器推送的物料包，支持审核、修改、发布

## 核心逻辑
1. **服务器推送** - WebSocket 推送每日物料包
2. **物料接收** - 客户端接收并保存到本地
3. **物料列表** - 显示最近N天的物料（待发布/已发布/草稿）
4. **物料详情** - 查看单个物料的完整内容
5. **编辑功能** - 修改标题、正文、图片、标签等
6. **发布操作** - 
   - 直接发布（调用MCP发布到小红书）
   - 保存草稿（不发布）
   - 定时发布（可选）

## 实现内容
1. 内容服务层
   - 接收WebSocket推送的物料
   - 本地CRUD操作
   - 发布状态管理
2. 内容IPC Handler
3. 内容管理界面
   - 物料日历视图（按日期展示）
   - 物料卡片（标题、封面、状态、操作按钮）
   - 物料编辑器（富文本、图片上传、标签）
   - 发布确认对话框
